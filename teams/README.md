# Class Webpage Project - Team Structure

## Overview

This semester, the class will collaboratively build a class webpage. The project is divided into **6 teams of 4 students each**, with each team responsible for a specific section of the website.

## Team Structure

### Team 1: Homepage/Landing Team
**Folder:** `homepage/`

**Responsibilities:**
- Design and implement the main landing page
- Create hero section with compelling visuals
- Develop site-wide navigation system
- Establish overall design system and visual identity
- Ensure responsive design and accessibility
- Create smooth user experience for site entry

**Key Deliverables:**
- Hero section with class branding
- Main navigation menu
- Design system/style guide (colors, typography, spacing)
- Responsive layout framework

---

### Team 2: About/Class Info Team
**Folder:** `about/`

**Responsibilities:**
- Create "About" section with course description
- Display instructor information and office hours
- Present learning outcomes clearly
- Design course overview and objectives
- Integrate class philosophy and approach
- Ensure information is accessible and well-organized

**Key Deliverables:**
- About page with course description
- Instructor profile section
- Learning outcomes display
- Course overview content

---

### Team 3: Schedule/Calendar Team
**Folder:** `schedule/`

**Responsibilities:**
- Build interactive schedule/calendar component
- Display class dates and important deadlines
- Highlight project due dates
- Create visual timeline of the semester
- Implement filtering/search functionality
- Make schedule mobile-friendly

**Key Deliverables:**
- Interactive calendar view
- Schedule table/list view
- Project deadline indicators
- Responsive schedule component

---

### Team 4: Student Work Showcase Team
**Folder:** `showcase/`

**Responsibilities:**

- Create project submission and display interface

- Ensure showcase is visually compelling

**Key Deliverables:**
- Project gallery grid/list view

- Media display components

### Team 5: Resources Team

## Collaboration Guidelines

### Design System
- All teams should follow a shared design system established by the Homepage team
- Use consistent colors, typography, spacing, and component styles
- Maintain visual cohesion across all sections

### Technical Standards
- Use semantic HTML5
- Follow responsive design principles (mobile-first approach)
- Ensure accessibility (WCAG guidelines)
- Use version control (Git/GitHub) for all code
- Document code and design decisions

### Communication
- Teams should communicate regularly about integration points
- Use GitHub Issues for tracking tasks and bugs
- Hold regular check-ins to ensure sections work together seamlessly
- Share progress and blockers in class

### Workflow
1. **Week 1-2:** Planning and design system establishment
2. **Week 3-4:** Initial prototypes and wireframes
3. **Week 5-6:** Development and implementation
4. **Week 7-8:** Integration and testing
5. **Week 9:** Final polish and launch preparation

---

## Team Roles (Within Each Team)

Each team of 4 should rotate through these roles:

1. **Design Lead** - Focuses on visual design, user experience, and aesthetics
2. **Development Lead** - Handles coding, technical implementation, and functionality
3. **Content Lead** - Manages text, copy, and information architecture
4. **Quality Assurance Lead** - Tests functionality, accessibility, and cross-browser compatibility

**Note:** Roles should rotate so each student experiences all aspects of web development.

---

## Integration Points

Teams need to coordinate on:
- **Navigation:** Homepage team creates nav, all teams integrate it
- **Design System:** Homepage team establishes, all teams follow
- **Footer:** Contact team creates, all teams integrate
- **Routing/Navigation:** Ensure smooth transitions between sections
- **Responsive Breakpoints:** Agree on consistent breakpoints
- **Color Palette:** Shared across all sections

---

## Success Criteria

The class webpage project will be considered successful when:
- ✅ All sections are complete and functional
- ✅ Design is cohesive across all sections
- ✅ Site is fully responsive and accessible
- ✅ All content is accurate and up-to-date
- ✅ Site is deployed and publicly accessible
- ✅ Code is well-documented and maintainable
- ✅ All students have contributed meaningfully
